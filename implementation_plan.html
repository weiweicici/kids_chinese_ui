Kids Chinese UI — 后端集成实施方案

目标设备: iPad Mini 1 · iOS 9.3.5 · armv7 · 512MB RAM · 零第三方库 · 2026-06-20

⚠ 核心约束
1. 只能用 NSURLSession + NSJSONSerialization（iOS 9 无原生 Codable / Combine）
2. 不能用 ZIP 解压库（armv7 无原生压缩 API，手写 zlib 极易 OOM）
3. JWT 必须存 Keychain（kSecClassGenericPassword），不能存 NSUserDefaults
4. 管理后台必须在 App 内实现，不用网页
5. 服务端 API 响应必须分页（LIMIT 20），防止大 JSON 撑爆 512MB
6. UI 布局代码不动

一、数据库 Schema

1.1 用户档案

CREATE TABLE profiles (
  id          uuid PRIMARY KEY REFERENCES auth.users(id),
  username    text UNIQUE NOT NULL,
  real_name   text,
  role        text CHECK (role IN ('student','teacher','admin')),
  is_approved boolean DEFAULT false,
  created_at  timestamptz DEFAULT now()
);

1.2 班级/师生绑定

CREATE TABLE class_assignments (
  id         uuid PRIMARY KEY DEFAULT uuid_generate_v4(),
  student_id uuid REFERENCES profiles(id),
  teacher_id uuid REFERENCES profiles(id),
  created_at timestamptz DEFAULT now()
);

1.3 学习进度

CREATE TABLE user_progress (
  id             uuid PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id        uuid REFERENCES auth.users(id),
  feature        text CHECK (feature IN ('main','game1','game2','game3','fullspell')),
  book_number    int,
  lesson_number  int,
  correct_count  int DEFAULT 0,
  total_attempts int DEFAULT 0,
  updated_at     timestamptz DEFAULT now(),
  UNIQUE (user_id, feature, book_number, lesson_number)
);

1.4 注册审批

CREATE TABLE registration_requests (
  id          uuid PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id     uuid REFERENCES auth.users(id),
  status      text CHECK (status IN ('pending','approved','rejected')),
  approved_by uuid REFERENCES profiles(id),
  created_at  timestamptz DEFAULT now()
);

1.5 课程发布

CREATE TABLE course_releases (
  id              uuid PRIMARY KEY DEFAULT uuid_generate_v4(),
  book_number     int,
  lesson_number   int,
  lesson_title    text,
  is_active       boolean DEFAULT true,
  plist_url       text,   -- Supabase Storage URL for session_{b}-{l}.plist
  audio_urls      jsonb,  -- {"1":"{url}","2":"{url}",...,"16":"{url}"}
  version         int DEFAULT 1,
  updated_by      uuid REFERENCES profiles(id),
  created_at      timestamptz DEFAULT now(),
  updated_at      timestamptz DEFAULT now()
);
设计说明：course_releases 表替代传统的 ZIP 包。每次 Admin 创建新课后，将 plist 和 MP3 逐个上传到 Supabase Storage，URL 存入此表。App 按需逐文件下载。

二、App 启动流程（AppDelegate.m 改造）

didFinishLaunchingWithOptions:
│
├─ Keychain 有 JWT?
│   ├─ NO  → rootVC = LoginViewController
│   └─ YES → GET /rest/v1/profiles?id=eq.{user_id}
│             │
│             ├─ student + is_approved == true
│             │   └─ rootVC = HomeScreenViewController
│             │
│             ├─ student + is_approved == false
│             │   └─ 弹窗 "等待老师/管理员审核" → 停留在 Login
│             │
│             └─ teacher / admin
│                 └─ rootVC = AdminViewController
│
└─ 后台启动课程检查（不影响 UI）
    GET /rest/v1/course_releases?version=gt.{localVersion}
    有新课程 → push 到下载队列（见第四章）
⚠ 关键注意：JWT 过期处理。每次 API 请求返回 401 时，不能自动刷新——iPad Mini 1 的 A5 单核芯片执行 Token 刷新请求期间 UI 会卡死。方案：返回 401 时直接退到 LoginViewController，让用户手动重新登录。

三、iOS 网络层（SupabaseClient.m）

3.1 架构

单例封装，全部手写 NSURLSession：

@interface SupabaseClient : NSObject
+ (instancetype)sharedClient;
- (void)GET:(NSString *)path completion:(void(^)(NSDictionary *, NSError *))completion;
- (void)POST:(NSString *)path body:(NSDictionary *)body completion:(void(^)(NSDictionary *, NSError *))completion;
- (void)downloadFile:(NSString *)url to:(NSString *)destPath completion:(void(^)(BOOL))completion;
- (void)saveToken:(NSString *)jwt;
- (NSString *)getToken;
- (void)clearToken;
@end

3.2 关键实现细节

  • Token 注入：每个请求在 URLRequest 的 setValue:forHTTPHeaderField: 自动添加 Authorization: Bearer <JWT>
  • Keychain 存储：SecItemAdd / SecItemCopyMatching 操作，使用 kSecClassGenericPassword + kSecAttrService:@"com.kidschinese.supabase"
  • 分页硬限：所有 GET 请求 URL 自动追加 &limit=20，防止大 JSON 撑爆内存
  • 内存安全：用 NSJSONSerialization JSONObjectWithData:options:NSJSONReadingMutableContainers 解析后立即释放 NSData（ARC 自动管理）

四、课程热更新 — 逐文件增量下载

核心原则：用磁盘空间换内存。NSURLSessionDownloadTask 流式写入 tmp/，全程 App RAM 零增长。

4.1 启动时对比版本

  1. App 启动时从 NSUserDefaults 读取本地 courseVersion
  2. GET /rest/v1/course_releases?version=gt.{localVersion} 拉取新增课程列表
  3. 无更新 → 跳过。有更新 → 进入下载流程

4.2 下载流程

下载队列控制器（串行，一次一个文件）:
│
├─ 1. 从 Documents/ 复制 chapter.plist（首次：从 Bundle copyItemAtPath:）
│
├─ 2. 对每一节新课，逐文件下载:
│    ├─ session_{b}-{l}.plist  → Documents/ChineseWordmp3/
│    ├─ {b}-{l}-{1..16}.mp3    → Documents/ChineseWordmp3/  (逐个)
│    └─ {b}-{l}.mp3             → Documents/ChineseWordmp3/
│
├─ 3. 每个文件:
│    ├─ NSURLSessionDownloadTask → tmp/ 路径
│    └─ moveItemAtURL:toURL:error:  → 目标路径（原子化移交）
│
├─ 4. 全部下载完成后:
│    ├─ NSMutableArray 读取 Documents/chapter.plist
│    ├─ 追加新 session 名
│    └─ writeToFile:atomically: 写回
│
└─ 5. [TextbookManager.sharedManager loadAllTextbooks];
    内存中的课程树自动刷新（resolvePath: 优先读 Documents，其次 Bundle）

4.3 删除课程

  1. Admin 在 Web 后台将 course_releases.is_active = false
  2. App 拉取到非活跃列表后，removeItemAtPath: 删除 Documents 下的对应文件
  3. 从 Documents/chapter.plist 移除该 session 条目
  4. 调用 loadAllTextbooks 刷新
⚠ 崩溃安全：下载过程中用户强退 App，下次启动时通过 moveItemAtURL 的原子性保证不会产生损坏文件。缺失的文件通过对比课程清单自动触发断点续传。

五、App 内管理后台（AdminViewController.m）

5.1 UI 架构

元素说明
UISegmentedControl顶部 Tab 切换：审批 | 学生进度 | 课程管理
UITableView (Cell 复用)每个 Tab 一个 TableView，dequeueReusableCellWithIdentifier: 保证内存安全
SquishyButton审批 Cell 内的「通过/拒绝」按钮，复用已有组件

5.2 Tab 1: 审批（Admin）

  • 拉取 registration_requests?status=eq.pending → 列表显示用户名 + 注册时间
  • 每个 Cell 两个按钮：「✅ 通过」(PATCH → is_approved=true, status=approved) 和 「❌ 拒绝」(status=rejected)
  • 操作后本地移除该行，不重新拉全量

5.3 Tab 2: 进度监控（Teacher/Admin）

  • 拉取 class_assignments?teacher_id=eq.{currentUserId} → 学生列表
  • 点击学生 → Push 到 StudentProgressViewController
  • 按课程分组展示 correct_count / total_attempts，以简单数字呈现，不用图表

5.4 Tab 3: 课程管理（Admin）

  • 拉取 course_releases 所有课程列表，显示 book/lesson/title/version/is_active
  • 每个 Cell 右侧 UISwitch 控制 is_active，切换时 PATCH 更新后端状态
  • 新增课程：Web 后台完成上传，App 端仅做开关控制
内存管理：didReceiveMemoryWarning 中清空非当前 Tab 的数据源数组（removeAllObjects），保留 TableView 骨架。不能 置空 self.view（iOS 9 不会自动重建子视图）。

六、离线优先与增量同步

6.1 策略

场景行为
学生玩游戏数据优先写入本地 NSUserDefaults（现有逻辑完全不变）
完成一节课saveGameProgress 后，将 {feature, book, lesson, correct, total} 压入本地上传队列
App 进入前台 / viewDidAppear检查上传队列，逐个 POST 到 /rest/v1/user_progress（UPSERT）
上传成功从队列移除；标记 NSUserDefaults 中该 key 为已同步
上传失败（无网络）保留队列，下次触发重试，不阻塞 UI

6.2 队列实现

用 NSMutableArray + NSUserDefaults 持久的简单队列（每项 < 1KB，安全）：

// 入队
[[NSUserDefaults standardUserDefaults] setObject:pendingUploads forKey:@"upload_queue"];
[[NSUserDefaults standardUserDefaults] synchronize];

// 出队（后台串行）
- (void)flushUploadQueue {
    NSMutableArray *queue = [self.uploadQueue mutableCopy];
    for (NSDictionary *item in queue) {
        [self POST:@"/rest/v1/user_progress" body:item completion:^(NSDictionary *res, NSError *err) {
            if (!err) [self.uploadQueue removeObject:item];
        }];
    }
}

七、RLS 策略（Supabase 数据库层权限）

所有权限在数据库层通过 Row Level Security 实现，App 端不需要额外逻辑：

-- Student: 只能读写自己的数据
CREATE POLICY student_own_data ON user_progress
  FOR ALL USING (auth.uid() = user_id);

-- Teacher: 能读自己班级学生的学习记录
CREATE POLICY teacher_view_students ON user_progress
  FOR SELECT USING (
    auth.uid() IN (
      SELECT teacher_id FROM class_assignments
      WHERE student_id = user_progress.user_id
    )
  );

-- Admin: 全局读写
CREATE POLICY admin_all ON user_progress
  FOR ALL USING (
    EXISTS (SELECT 1 FROM profiles WHERE id = auth.uid() AND role = 'admin')
  );
